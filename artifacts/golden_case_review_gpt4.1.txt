## 1. **Coverage Gaps & Missing Edge Cases**

### **A. General Gaps Across All Tasks**
- **Required vs. Optional Fields**: No explicit tests for missing required fields (e.g., missing `name` or `email` for clients, `amount` for opportunities/quotes).
- **Null/Empty Values**: No cases for fields being `null`, empty strings, or whitespace (e.g., `"name": ""` or `"email": null`).
- **Field Length/Size Limits**: No tests for excessively long strings (e.g., 256+ chars in `name`, `notes`, `file_name`).
- **Special Characters/Unicode**: No cases with special characters, emojis, or non-ASCII in names, notes, or file names.
- **Case Sensitivity in Enums**: Only lowercase rejection is tested; mixed-case (e.g., `"Active"`, `"aCtIvE"`) is not.
- **Date Format Robustness**: Only ISO format is used; no tests for alternate formats, invalid dates, or past/future logic.
- **Numeric Boundaries**: No tests for zero values (e.g., `amount: 0`), or for maximum allowed values (e.g., `amount: 1e9`).
- **Boolean/Flag Fields**: If any exist in the schema, not covered.

### **B. Task-Specific Gaps**

#### **create_new_client**
- **Duplicate Name, Different Email**: Only duplicate email is tested.
- **Industry/Owner/Address with Invalid Types**: No tests for non-string types (e.g., `industry: 123`).
- **Phone Format**: Only valid phone numbers are present; no tests for invalid formats or country codes.
- **Status Enum**: Only lowercase rejection; no test for invalid status (e.g., `"Pending"`).

#### **create_new_opportunity**
- **Probability Edge Cases**: Only >100 and <0 are tested; no test for exactly 0 or 100.
- **Stage Regression/Progression Rules**: No explicit test for invalid stage transitions (e.g., skipping stages).
- **Amount as String/Non-numeric**: No test for `"amount": "10000"` or `amount: null`.
- **Owner/Notes as Non-string**: Not tested.

#### **create_quote**
- **Status Enum**: Only lowercase rejection; no test for invalid status (e.g., `"InReview"`).
- **Version Format**: No test for invalid version formats (e.g., `"version": 3` or `"version": "v1.0.0"`).
- **valid_until in the Past**: No test for expired quotes.
- **Amount as Zero or Excessively Large**: Not tested.

#### **upload_document**
- **File Name Edge Cases**: No test for empty, very long, or invalid file names (e.g., `"file_name": ""`, `"file_name": "a"*300+".pdf"`).
- **Entity Type Enum**: Only lowercase rejection; no test for invalid entity types (e.g., `"entity_type": "Deal"`).
- **File Extension Validation**: No test for unsupported extensions (e.g., `.exe`, `.bat`).

#### **modify_opportunity**
- **Partial Updates**: No test for empty `updates` dict.
- **Multiple Invalid Updates**: No test for multiple errors in one update (e.g., invalid stage and negative probability).
- **Immutable Fields**: Only unsupported field tested; not fields that should be immutable (e.g., `opportunity_id`).

## 2. **Ambiguous or Brittle Assumptions**

- **Casing**: Only lowercase is rejected; mixed-case and leading/trailing whitespace are not tested.
- **Date Formats**: Only `"YYYY-MM-DD"` is used; no test for `"MM/DD/YYYY"`, `"YYYY/MM/DD"`, or invalid dates (`"2025-02-30"`).
- **Probability**: Assumed integer; no test for float (e.g., `probability: 33.3`), string (`"33"`), or missing probability.
- **Email Validation**: Only malformed email tested; no test for valid but unusual emails (e.g., `"user+tag@example.com"`).
- **ID Placeholders**: All use `<derived_id>`; no test for invalid, empty, or non-string IDs.
- **Enum Values**: Only one invalid value per enum tested; not comprehensive.
- **Field Optionality**: Not clear which fields are required vs. optional in negative cases.

---

## 3. **Opportunities to Mirror Deterministic Customer Checks**

- **Explicit Required Field Checks**: Add cases where required fields are missing or null.
- **Strict Enum Enforcement**: Add tests for mixed-case, whitespace, and invalid enum values.
- **Type Enforcement**: Add cases where fields are of the wrong type (e.g., string instead of float).
- **Date Logic**: Add checks for past/future constraints, invalid dates, and alternate formats.
- **Relationship Enforcement**: Add tests for referencing deleted or archived entities, not just unknown ones.
- **Boundary Testing**: Add min/max/zero values for numerics and string lengths.
- **Immutability**: Test attempts to modify immutable fields (e.g., IDs).
